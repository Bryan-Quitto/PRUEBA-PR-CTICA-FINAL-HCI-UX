# Sprint Planning — Mejoras UX Críticas

**Sprint:** 1 (Prueba Final)
**Duración:** 2 horas (Simulado)
**Objetivo del Sprint:** Rediseñar el Dashboard Global para mejorar la jerarquía visual e implementar un sistema de feedback visual dinámico para mejorar la comunicación sistema-usuario.

## Sprint Backlog (Tareas Seleccionadas)

1.  **Evaluación Heurística Inicial:** Identificar 10 problemas críticos en el Dashboard actual para guiar el rediseño. (Fase 2)
2.  **Diseño de Wireframes en HTML:** Crear prototipos de baja, media y alta fidelidad para validar la nueva arquitectura del Dashboard. (Fase 3)
3.  **Implementación de Feedback Visual:** Desarrollar e integrar un sistema de notificaciones (Toasts) en las interacciones principales del Dashboard. (Fase 4)
4.  **Documentación de Evidencia IA:** Registrar los prompts y el apoyo de la IA en el proceso de diseño y desarrollo. (Fase 5)

## Definición de Hecho (DoD)
*   El código compila sin errores.
*   La UI sigue los principios de Gestalt y jerarquía visual.
*   El feedback visual se activa en al menos 3 interacciones clave.
*   Documentación completa en la carpeta `HCI-PruebaFinal/`.
*   Commits realizados con mensajes técnicos.

## Capacidad del Equipo
*   **UX Engineer (Gemini CLI):** 100% de dedicación.
*   **Stakeholder (Estudiante):** Validación de avances.
