# Product Backlog — Usability Test Dashboard 2.0

Este backlog contiene las necesidades de usuario identificadas para mejorar la experiencia, navegación y retroalimentación del sistema, priorizadas para la Prueba Práctica Final de HCI/UX.

| ID | Historia de Usuario | Prioridad | Estimación (Story Points) |
| :--- | :--- | :--- | :--- |
| **US-01** | **Rediseño Estructural del Dashboard Global:** Como administrador, quiero una visualización clara y jerarquizada de las métricas para tomar decisiones rápidas sin sobrecarga cognitiva. | Alta | 8 |
| **US-02** | **Feedback Visual Dinámico (Toasts):** Como usuario, quiero recibir confirmaciones visuales inmediatas al realizar acciones (filtros, guardar, errores) para saber que el sistema procesó mi petición. | Alta | 5 |
| **US-03** | **Prevención de Errores en Formularios:** Como evaluador, quiero que el sistema me advierta antes de cometer errores críticos en la carga de datos para asegurar la integridad de los reportes. | Media | 5 |
| **US-04** | **Arquitectura de Información y Navegación:** Como usuario nuevo, quiero una navegación intuitiva y contextual (breadcrumbs/menús claros) para no perderme entre las diferentes vistas del aplicativo. | Alta | 3 |
| **US-05** | **Diseño Emocional y Estética:** Como usuario recurrente, quiero una interfaz moderna y agradable que reduzca el estrés visual durante el análisis de usabilidad. | Baja | 2 |

---
**Nota:** El enfoque de este Sprint se centrará en el rediseño del Dashboard y el Feedback Visual.
